## Overview

This project applies the four pillars of object-oriented programming to a structured group assignment. The system models a real-world domain using a hierarchy of classes, abstract types, and interfaces, with clean separation between domain logic and console interaction.

## OOP Concepts Demonstrated

| Concept | How it's used |
|---|---|
| **Encapsulation** | Private fields with public getters/setters; validation inside setters. |
| **Inheritance** | Base class extended by specialized subclasses sharing common behaviour. |
| **Polymorphism** | Overridden methods invoked through base-class references. |
| **Abstraction** | Abstract classes and interfaces define contracts for concrete implementations. |
| **Composition** | Aggregate types built from smaller domain classes. |
